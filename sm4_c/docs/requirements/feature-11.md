# Feature-11: Base64 解码非法字符校验

## 优先级: 高

## 问题描述

`base64_decode()` 函数（sm4_ext.c:699-747）使用查找表将 Base64 字符映射到 0-63 的值，非法字符被映射为 64。但代码未检查返回值是否为 64，导致：

1. 非法字符（如空格、特殊符号）被静默当作有效字符解码
2. 攻击者可构造恶意 Base64 字符串产生任意字节输出
3. 在 GCM 场景中，可能绕过 Tag 验证或产生意外的解密结果

### 漏洞代码

```c
// base64_decode_table 中非法字符映射为 64
val |= base64_decode_table[(uint8_t)input[i]] << 18;  // 64 << 18 = 越界值
```

## 影响范围

- `sm4_ext.c:699-747` — base64_decode 函数
- `sm4_ext.c:865-985` — sm4_decrypt_gcm_base64 函数（依赖 base64_decode）

## 需求描述

1. 在解码循环中检查每个查表结果，若为 64 则拒绝输入
2. 忽略 `=` 填充字符（已在代码中处理）
3. 对非法 Base64 输入返回明确错误

## 验收标准

1. 非法 Base64 字符串（如 `"ab!d"`）被拒绝并返回错误
2. 合法 Base64 字符串（包括含 `=` 填充的）被正确解码
3. sm4_decrypt_gcm_base64 对非法输入返回错误而非静默处理
