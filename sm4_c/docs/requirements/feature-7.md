# Feature-7: hex_to_bytes 错误处理增强

## 优先级: 中

## 问题描述

`hex_to_bytes()` 函数使用 `strtol()` 将十六进制字符转换为字节，但未检查转换错误。`strtol` 在遇到非法字符时返回 0 且不设置错误指示，导致：

1. 非法十六进制字符（如 "gg"、"zz"）被静默转换为 0
2. 无法区分合法的 "00" 和非法输入
3. 可能导致密钥或 IV 被错误解析

## 影响范围

- `sm4_ext.c:29-47` — hex_to_bytes 函数
- 所有使用十六进制密钥/IV 的加密解密函数

## 需求描述

1. 使用 `strtoul()` 替代 `strtol()`，并检查 `endptr` 参数
2. 验证每个十六进制字符是否在 `0-9a-fA-F` 范围内
3. 对非法输入返回明确的错误码
4. 在调用方添加错误检查

## 参考代码位置

- `sm4_ext.c:29-47` — hex_to_bytes 函数
- `sm4_ext.c:63-85` — get_key_bytes 函数

## 验收标准

1. 非法十六进制字符串被正确拒绝并返回错误
2. 合法十六进制字符串（包括 "00"）被正确解析
3. 所有调用方正确处理 hex_to_bytes 的错误返回
