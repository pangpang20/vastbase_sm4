# Feature-13: PBKDF2 迭代次数提升

## 优先级: 中

## 问题描述

`derive_key_and_iv()` 函数（sm4.c:673-719）中 PBKDF2 迭代次数硬编码为 10000 次：

```c
int iterations = 10000;
```

OWASP 2023 密码存储指南推荐 PBKDF2-SHA256 至少 600,000 次迭代。10000 次迭代使密码派生的密钥容易被暴力破解。

## 影响范围

- `sm4.c:684` — derive_key_and_iv 中的 iterations 变量
- 所有使用 KDF 功能的加密/解密路径

## 需求描述

1. 将默认迭代次数提升至 600,000 次（或至少 100,000 次）
2. 考虑允许通过参数指定迭代次数（向后兼容）
3. 在文档中说明迭代次数与安全性的关系

## 验收标准

1. 默认迭代次数不低于 100,000 次
2. 使用新迭代次数派生的密钥与旧密钥不兼容（需明确标注版本差异）
3. 性能在可接受范围内（单次派生 < 1 秒）
