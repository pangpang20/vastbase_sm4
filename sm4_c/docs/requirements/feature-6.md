# Feature-6: PKCS7 填充预言攻击防护

## 优先级: 中

## 问题描述

CBC 模式解密中的 PKCS7 去填充实现存在填充预言攻击风险。当前实现对填充无效和解密失败返回相同的错误码（-1），攻击者可通过观察错误差异推断明文。

具体问题：
1. `pkcs7_unpad()` 填充校验失败返回 -1
2. 解密函数对填充失败和其他失败返回相同错误
3. 错误消息未区分填充错误和认证错误

## 影响范围

- `sm4.c:194-216` — pkcs7_unpad 函数
- `sm4.c:254-281` — sm4_ecb_decrypt 函数
- `sm4.c:326-362` — sm4_cbc_decrypt 函数

## 需求描述

1. 在 CBC 解密中，填充失败不应产生可区分的错误响应
2. 考虑使用 "先验证后解密" 或 "恒定时间去填充" 技术
3. 错误消息统一为 "decryption failed"，不泄露具体失败原因
4. 如果可能，使用 Encrypt-then-MAC 模式替代裸 CBC

## 参考代码位置

- `sm4.c:194-216` — pkcs7_unpad 函数
- `sm4.c:356-361` — CBC 解密中的填充校验

## 验收标准

1. 填充错误和解密错误返回相同的错误消息
2. 去填充操作使用恒定时间实现
3. 无法通过错误响应区分填充无效和密文无效
